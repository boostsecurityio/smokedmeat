# Release Prep And Launch Materials

## Why This Exists

SmokedMeat is targeting a public open source release on April 15, 2026.

After feature freeze, the remaining work is not feature delivery. The focus shifts to making the release understandable, teachable, and publishable.

This task groups the post-freeze work into one place so it does not get scattered across ad hoc notes and last-minute checklists.

## Release Window

- Feature freeze target: April 10, 2026
- Pre-freeze rule: only take work that can ship confidently in small chunks before the freeze date
- Post-freeze working days: April 10, 2026 and April 13-14, 2026
- Public release target: April 15, 2026

## Scope

This task covers:

- tutorial and walkthrough content
- screenshots and screencast capture
- release notes
- blog article inputs
- packaging and release verification

This task does not cover new feature work unless a blocker is discovered during release prep.

## Deliverables

### 1. Tutorial

Prepare a concise end-to-end tutorial that matches the product state being released.

Minimum expectations:

- setup path
- target analysis
- vuln selection
- exploit execution
- callback / dwell / post-exploit flow
- pivot examples where appropriate

### 2. Walkthrough Visuals

Prepare the visual assets needed for release and training material.

Minimum expectations:

- clean screenshots
- at least one screencast or walkthrough capture
- stable demo path that matches the documented tutorial

### 3. Release Notes

Summarize what is shipped, what is intentionally out of scope, and what known rough edges remain.

### 4. Blog Article Inputs

Prepare the technical inputs needed for the launch article:

- supported workflows
- core operator story
- notable differentiators
- realistic screenshots and short demos

### 5. Packaging And Verification

Confirm the public release artifacts and instructions are coherent.

Minimum expectations:

- release packaging is verified
- quickstart or onboarding instructions are current
- docs and screenshots match the shipped product

## Done Criteria

This task is in good shape when:

- the release tutorial is ready
- screenshots and screencast assets are ready
- release notes are ready
- the blog article has the inputs it needs
- packaging and onboarding are rechecked against the release build
